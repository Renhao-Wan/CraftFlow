# CraftFlow 未来拓展路线图

> 本文档规划了 CraftFlow 从当前原型演进为生产级平台的功能方向与架构演进路径。

## 演进背景

当前 CraftFlow 已完成核心功能验证，实现了基于 LangGraph 的双图架构（Creation Graph + Polishing Graph）和基础的前后端交互。为进一步提升产品竞争力和用户体验，规划以下三个维度的功能拓展。

---

## 一、Python 后端（AI 层）— 专注 AI 与 LangGraph 执行

AI 层聚焦于提升生成质量和扩展创作能力，不承载业务逻辑。

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| 创作模板系统 | 预设不同类型的创作图（论文、小说、技术文档），每种模板对应不同的 Graph 配置 | 中 |
| 多轮润色 | 支持用户对结果进行二次/三次润色，形成润色链 | 低 |
| 流式输出 | 创作过程中实时推送生成的段落，而非完成后一次性返回 | 中 |
| RAG 增强 | 用户上传参考文档，创作时检索相关内容辅助生成 | 高 |
| 自定义 Prompt | 允许用户自定义写作风格、语气等 Prompt 参数 | 低 |

---

## 二、前端（交互层）— 面向用户体验优化

前端以提升交互流畅度和降低使用门槛为目标。

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| 暗黑模式 | CSS 变量切换深色主题 | 低 |
| 草稿自动保存 | 输入内容存 localStorage，刷新不丢失 | 低 |
| 任务搜索/筛选 | 按类型、状态、关键词筛选历史任务 | 低 |
| 批量操作 | 批量删除、批量导出任务 | 低 |
| 结果对比视图 | 创作任务支持大纲与最终结果的对比查看 | 中 |
| 流式渲染 | 配合后端流式输出，逐段显示生成内容 | 中 |
| 快捷键支持 | Ctrl+Enter 提交、Esc 返回等 | 低 |
| 字数/阅读时间统计 | 结果页显示字数、段落数、预计阅读时间 | 低 |

---

## 三、Java 后端（业务层）— 承载核心业务逻辑

业务层负责用户管理、数据持久化和商业化能力，与 AI 层解耦。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 用户系统 | 注册、登录、JWT 认证、个人资料 | 高 |
| 任务持久化 | 替代当前 SQLite，使用 MySQL/PostgreSQL 存储任务数据 | 高 |
| 任务权限 | 用户只能访问自己的任务，支持分享链接 | 高 |
| 配额管理 | 每日创作/润色次数限制、会员等级 | 中 |
| 模板市场 | 用户创建、分享、使用大纲模板 | 中 |
| 数据统计 | 使用趋势、任务完成率、AI 调用统计 | 中 |
| 通知系统 | 任务完成推送、系统公告 | 低 |
| 支付/会员 | 付费功能、订阅管理 | 低 |
| API 网关 | 对外提供 REST API，支持第三方接入 | 低 |

---

## 架构设计核心原则

- **Python 后端**：只做 AI 相关能力，不承载业务数据、用户、权限
- **Java 后端**：统一管理用户、数据、权限、配额、商业化
- **前端**：对接两套后端，统一呈现给用户
